# Supabase (Cloud SQL) Setup Guide

This guide explains how to set up a free cloud PostgreSQL database on Supabase and connect it to your guest logging web application.

---

## Step 1: Create a Free Supabase Project

1. Go to [supabase.com](https://supabase.com) and click **Start your project** or **Sign In**. You can sign in instantly using your **GitHub account** for free.
2. In the dashboard, click **New Project** and select your organization.
3. Fill in the project details:
   - **Name**: `Guest Database` (or any name you like).
   - **Database Password**: You can use the password you generated, e.g., `AA9C6BEEFD6E2193C7949AC6FF9E2614`.
   - **Region**: Choose the region closest to you.
   - **Pricing Plan**: Choose **Free** (no credit card required).
4. Click **Create new project** and wait a couple of minutes for your database to be provisioned.

---

## Step 2: Create the SQL Database Table

Once your project is ready:
1. In the left sidebar of the Supabase dashboard, click on the **SQL Editor** icon (looks like `>_` or SQL terminal).
2. Click **New Query** (or **New tab**).
3. Paste the following SQL script into the editor:

```sql
-- Create the guests table
create table guests (
  id bigint generated by default as identity primary key,
  name text,
  dob text,
  birthdate_correct_up_to text,
  gender text,
  nationality_code text,
  passport_number text,
  arrival_date text,
  expected_leaving_date text,
  checkout_date text,
  room_number text,
  logged_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- Enable Row Level Security (RLS)
alter table guests enable row level security;

-- Create Policies to allow public access (useful for static github pages)
create policy "Allow public read access" 
on guests for select 
using (true);

create policy "Allow public insert access" 
on guests for insert 
with check (true);

create policy "Allow public delete access" 
on guests for delete 
using (true);
```

4. Click the **Run** button (bottom right or top right). You should see a success message (`Success. No rows returned`).

---

## Step 3: Connect the Web App to Your Database

1. In the left sidebar, click the **Project Settings** gear icon, then go to **API**.
2. Under **Project API keys**, copy the following two values:
   - **Project URL** (under the `Project URL` header).
   - **Anon Public Key** (under the `Project API keys` header, marked as `anon` `public`).
3. Open [assets/js/db.js](file:///d:/uni/2025.1/web/prjjjj/hanguyen225.github.io/assets/js/db.js) in your text editor.
4. Replace the empty strings at the top of the file with your copied values:
   ```javascript
   const SUPABASE_URL = "https://your-project-id.supabase.co";
   const SUPABASE_ANON_KEY = "your-anon-public-key-here";
   ```
5. Save the file and publish/push your code.

---

## Testing & How It Works

* **Automatic Switching**: The application will automatically check if the `SUPABASE_URL` and `SUPABASE_ANON_KEY` are configured in `db.js`.
* **Fallback**: If they are empty, it will continue using your local browser **IndexedDB** database.
* **Cloud SQL**: If they are filled, it will automatically connect to your hosted database on **Supabase**. Anyone visiting your website will view and log guests to the same cloud SQL table!
